# Sliect Launcher 更新日志

## v2.2.260716.1435

### 新功能

- **服务器/远程桌面模式**
  - 新增环境变量 `SLIECT_SERVER_MODE=1`，在虚拟服务器或远程桌面中自动：
    - 跳过 GDI 启动画面
    - 禁用 WebView2 GPU 加速
    - 禁用渲染器代码完整性检查
    - 注入 WebView2 兼容性启动参数（`--disable-gpu` 等）
  - 新增 `SLIECT_NO_SPLASH=1` 快速跳过启动画面

- **WebView2 运行环境检测**
  - 启动前检测 WebView2 Runtime 是否安装
  - 检测方式：注册表 + 文件系统双保险
  - 未检测到时弹出原生提示，提供「继续尝试/退出」选项，避免误报阻断启动
  - 环境检测在启动画面显示后执行，不阻塞窗口弹出
  - `SLIECT_SKIP_ENV_CHECK=1` 用于 CI/构建环境跳过检测

- **启动画面 (Splash) 增强**
  - 新增 GDI+ 原生启动画面 `splash_windows.go` + `splash_native.c`
  - 自适应文字大小，以窗口高度 240px 为基准缩放
  - 显示启动进度条和当前执行步骤
  - 版本号右下角显示
  - 启动画面 800ms 后自动取消置顶，避免挡住后续 MessageBox/确认框

- **启动进度优化**
  - 进度条从 `main.go` 到 `app.startup` 连续推进，不再重置
  - 每一步操作都有对应进度和状态文字
  - 自启项目按实际数量分配进度
  - 每步之间增加 5ms 延迟，让进度变化肉眼可见

### 修复

- 修复启动画面在虚拟服务器/远程桌面中因窗口创建失败、消息循环挂起或 WebView2 初始化挂起而卡住的问题
  - `showSplash()` 10 秒超时
  - `closeSplash()` 5+2 秒超时
  - `main()` 20 秒看门狗，检测到进度推进时自动重置
- 修复版本号显示位置偏移问题
- 修复启动画面文字被进度条遮挡/裁切的问题
- 修复启动画面文字过小的问题

### 其他

- 通知中心滚动提示优化（显示时间、位置调整）
- 日志缓冲区新增累计级别统计
